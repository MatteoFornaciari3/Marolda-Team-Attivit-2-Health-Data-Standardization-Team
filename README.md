# Alma FHIR Mapping Pipeline

## Overview

This repository provides a modular pipeline for transforming clinical research datasets into [HL7 FHIR](https://www.hl7.org/fhir/) format. It includes optional preprocessing, data transformation, FHIR resource generation, and validation steps.

The pipeline supports multiple studies or projects, each housed in a dedicated subdirectory under `studies/`.

## Project Structure

```bash
.
├── studies/
│   └── <study-name>/
│       ├── 00-input/                # Raw clinical data (CSV files). [User provided]
│       ├── 01-refinement/
│       │   ├── clean/               # Cleaned CSVs. [Auto-generated]
│       │   ├── json/                # Converted JSON files. [Auto-generated]
│       │   └── scripts/             # Optional preprocessing scripts:
│       │       ├── data_cleaning.ipynb
│       │       ├── data_cleaning.types.json
│       │       └── data_cleaning.whitelist.json
│       ├── 02-mapping/
│       │   ├── indexes/             # Index configs for csv-to-json & conversion. [User provided]
│       │   ├── mappings/            # StructureMap files for data transformation. [User provided]
│       │   ├── resources/           # (Optional) FHIR resources to pre-upload (CodeSystems, etc.). [User provided]
│       │   └── structures/          # FHIR StructureDefinition resources defining input structure. [User provided]
│       ├── 03-publishing/
│       │   └── bundles/             # FHIR Bundles generated by Matchbox. [Auto-generated]
│       └── 04-validation/
│           └── validation-queries-test.ipynb  # Optional validation notebook.
├── shared/
│   ├── bulk-data-export.py         # Export .ndjson from a FHIR server
│   ├── clean.sh                    # Utility to clean generated files
│   └── create-study.sh             # Scaffolds a new study directory
├── docker-compose.yaml             # Setup for HAPI FHIR and Matchbox
├── main.sh                         # Main orchestration script (per study)
├── requirements.txt                # Python dependencies
└── README.md                       # You’re here.
````

## Getting Started

### 1. Clone the Repository

```bash
git clone https://github.com/your-org/clinical-data-to-fhir-pipeline.git
cd clinical-data-to-fhir-pipeline
```

### 2. Create a New Study

```bash
./shared/create-study.sh <study-name>
```

> This scaffolds the folder structure under `studies/<study-name>/`.

### 3. Add Your Study Data and Configuration

* **Raw CSVs** → put into `00-input/`
* **Index Configs** → fill in `csv-to-json-index.json` and `conversion-index.json` under `02-mapping/indexes/`
* **StructureMap Files** → add to `02-mapping/mappings/`
* **(Optional) FHIR Resources** → add any base resources (e.g., `CodeSystem`, `ValueSet`, `Questionnaire`) to `02-mapping/resources/` to pre-load them into the FHIR server
* **StructureDefinitions** → place FHIR `StructureDefinition` resources into `02-mapping/structures/`. These describe the expected structure of the input data and are uploaded to the Matchbox server
* **(Optional) Preprocessing Scripts** → customize the notebook and JSON configs under `01-refinement/scripts/` if you need to preprocess or filter input data

### 4. Set Up Python Environment

```bash
python3 -m venv .venv
source .venv/bin/activate
pip install -r requirements.txt
```

### 5. Install FHIR Pipeline Utils

Install and configure [FHIR Pipeline Utils](https://gitlab.com/almahealthdb/fhir-pipeline-utils#usage) as per their documentation.

### 6. Start the Required Services

```bash
docker-compose up
```

This starts:

* A [HAPI FHIR Server](https://hapifhir.io/)
* A [Matchbox Server](https://github.com/ahdis/matchbox) for resource transformation


## Running the Pipeline

To process a specific study:

```bash
./main.sh <study-name>
```

This script will:

1. Optionally run the data cleaning/preprocessing notebook
2. Convert cleaned CSVs into JSON
3. Upload `StructureDefinitions` to Matchbox
4. Optionally upload base FHIR resources to the FHIR server
5. Upload mapping definitions to Matchbox
6. Transform the JSON data into FHIR Bundles
7. Send the Bundles to the FHIR server


## Exporting FHIR Resources

To export FHIR data as `.ndjson` for backup or analysis:

```bash
python shared/bulk-data-export.py <FHIR_SERVER_URL> <OUTPUT_FOLDER>
```

* `<FHIR_SERVER_URL>`: URL of the FHIR server (e.g., `http://localhost:8081/fhir`)
* `<OUTPUT_FOLDER>`: Destination directory for exported files


## Notes

* `clean.sh` removes all generated data except raw input
* Studies are isolated and reusable
* `validation-queries-test.ipynb` can be customized to run data consistency checks post-conversion
